Portion Calculator

An easy to use tool for calculating the cost of a portion or recipe

View Github

View Project
  • HTML
  • CSS
  • Vanilla Javascript

After finishing my Hangman game I wanted to build something with utility, something that I would find useful. At the time, I was looking into food prepping and wanted an easy way to calculate the cost and nutritional value of, for example, 567 grams of lasagne out of a 3468-gram batch. My original idea was to build something similar to a calorie tracking or meal prepping app that would also be able to price up the meals. I later decided to drop the calorie counting functionality because I didn’t continue with my meal prepping. This made it much less important, and besides, there are so many calorie-tracking apps out there. There weren’t however many that could calculate the cost of a recipe as I could only find one website that kind of did food cost calculations the way I was envisioning it, and I thought I could do better.

My goal was to build something simple, which was another reason for dropping the calorie-tracking functionality, and something other people could use. I intentionally kept color out of my previous project so this time I wanted to create a colorful UI. I wanted to work on my CSS skills, specifically the layout, and I wanted it to look like a dashboard or application instead of a generic website.

Challenges

The time at which this project took place is twofold as I started the project but didn’t finish it because at the time I was moving from project to project trying to learn as many things as I could. I came back to it just under a year later and I found it very difficult and very overwhelming at first because there was so much that I wanted to change and I didn’t understand how the code worked anymore. I found that simply writing a list of tasks that I could work through one by one allowed me to solve the problems quickly. I ended up almost rewriting the entire HTML and CSS code, which involved applying my own mini CSS flex framework, changing class names and selectors, and designing a completely fresh UI. I didn’t remove too much from my JS.app besides refactoring all of my DOM selectors and adding in a couple of new best practices I had learned, I mostly just build on top of it.

I drew inspiration for the UI from Toggl’s time tracking app and various dashboard templates I found online. I wanted something smart yet playful, and something that allowed me to be creative and try out novel features, such as the sidebar and hamburger button, maybe even unconventional ways of doing things, for learning’s sake. One of the biggest challenges I came across was with the side menu. I knew how to make a simple side menu move in and out but I wanted this one to look like the main content was being pulled across to reveal the menu that was lying beneath. Similar to the kind of effect you get with some apps such as Discord on Android. I came up with a few difficult solutions before finding a relatively simple way. There is probably a reason why I don’t think I’ve seen this kind of approach done on a desktop web app before, and it’s because it squishes the content when it opens and is overly complicated from a cost-benefit perspective. However I think it looks cool, I learned some things about “z-index’s” I didn’t know and also some things that don’t work and that I shouldn’t do in the future.
